Pado Desktop

PADO Desktop on Web is launched by JNLP. With the proper security settings, JNLP launched applications can run just like any stand-alone applications with full access to the local file system and network. PADO Desktop on Web is configured as such and has the same look and feel of the stand-alone PADO Desktop.

Launching PADO Desktop for the First Time

The following steps must be taken before you can successfully launch PADO Desktop:
  1. Download and save the self-signed certificate in your local file system by clicking here: Netcrest Trusted Certificate
  2. Import the self-signed security certificate from Java Control Panel
    • Open Java Control Panel. On Windows, select "Java" in Control Panel. On Mac, select "Java" in System Preferences.
    • Select the "Security" tab.
    • Select the "Manage Certificates..." button.
    • Select "Signer CA" from the combobox.
    • Select the "Import" button.
    • Import the downloaded certificate.
  3. Launch PADO Desktop by clicking here: Launch Pado Desktop

Desktop Login

PADO requires you to login with proper credentials before you can access the grids. The login window lists the available grid environments in the "Env" combobox. The "_USER" environment is for you to enter locators of your choice in the "Locators" text field. The other environments listed in the combobox are pre-configured for you and their locators cannot be changed.

Desktop Template

When you launch PADO Desktop for the first time, you may see a blank desktop. You can start laying out the screen by splitting a worksheet into "quads" and dropping components from the Bean Bar into the empty quads. An easier way is to open the pre-built template that has most of the essential components to monitor and control the PADO grids as follows:
  • From the menubar, Select File/Open Template

Reinstalling PADO Desktop

At times, JNLP may not launch PADO Desktop due to the staled resources. You can reinstall PADO Desktop by deleting the Java Cache as follows:
  • Open Java Control Panel - On Windows, this is found in Control Panel. On Mac, this is found in System Preferences.
  • From the "General tab, select "Settings..."
  • From the "Temporary Files Settings" window, select "Delete Files..."
  • To reinstall, select "Installed Applications and Applets" and then the "OK" button.
